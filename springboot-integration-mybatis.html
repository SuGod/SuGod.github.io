在传统项目中使用mybatis需要很多配置文件,但是在springboot项目中,一切都化繁为简.

下面我们以传统xml方式和配置方式分别来配置 mybatis在springboot 中应用.

创建springboot工程,在pom.xml中加入mybatis依赖;

 <dependency>
    <groupId>org.mybatis.spring.boot</groupId>
    <artifactId>mybatis-spring-boot-starter</artifactId>
    <version>1.3.1</version>
</dependency>

在application.properties 或 yml 中添加如下配置:

##数据库连接信息
spring.datasource.url=jdbc:mysql://localhost:3306/test_db?characterEncoding=utf8&useSSL=true
spring.datasource.username=root
spring.datasource.password=root
spring.datasource.driver-class-name=com.mysql.jdbc.Driver

spring:
    datasource:
        name: test
        url: jdbc:mysql://localhost:3306/test_db?characterEncoding=utf8&useSSL=true
        username: root
        password: root
        # 使用druid数据源
        type: com.alibaba.druid.pool.DruidDataSource
        driver-class-name: com.mysql.jdbc.Driver

springboot 会自动加载 配置文件中的 spring.datasource.* 相关配置,把datasource注入到sqlSessionFactory中,然后把sqlSessionFactory注入到Mapper中.接下来就可以直接使用了.

在启动类 Application 中添加注解 @MapperScan.

@SpringBootApplication
@MapperScan("com.zhangdi.mapper")//扫描mapper下所有的类
public class Application{

    public static void main(String[] args){
        SpringApplication.run(Application.class,args);
    }
}

基于注解方式实现增删改查

public interface UserMapper{

    @Select("SELECT * FROM user")
    @Results(id = "userMapper"
            value = {
                @Result(property = "username",column = "user_name"),
                @Result(property = "nickname",column = "nick_name")
            }
    )
    List<User> findAll();

    @Select("SELECT * FROM user WHERE id = #{id}")
    @Results(
        /**
        *property 实体字段, column 数据库表字段
        * 一个 @Result 表示 实体和数据库表的映射关系
        * 实体字段和数据库字段不一样 需要 描述映射关系,如果一样可以省略.
        */
        @Result(property = "username",column = "user_name"),//jdbcType = "",javaType = ""  具体类型暂时省略 后面会说.
        @Result(property = "nickname",column = "nick_name")
    )

    @Select("SELECT * FROM user WHERE id = #{username}")
    @ResultMap("userMapper")
    User findByUsername(String username);

    @Insert("INSERT INTO user(user_name,password,nick_name,age) VALUES(#{username},#{password},#{nickname},#{age})")
    void insert(User user);

    @Update("UPDATE user SET user_name=#{username},nick_name=#{nickname},age=#{age}  WHERE id = #{id}")
    void update(User user);

    @Delete("DELETE FROM user WHERE id =#{id}")
    void delete(Integer id);
}

首先说 @Result,以上代码可以看出,没有给 @Results 设置id看属性需要给所有方法都添加一遍 映射关系配置,代码冗余.所以添加id 并使用@ResultMap 实现代码复用;

@Insert
@Update
@Delete
@Select

这些注解中的每一个代表了执行的真 实 SQL。 它们每一个都使用字符串数组 (或单独的字符串)。如果传递的是字符串数组, 它们由每个分隔它们的单独 空间串联起来。这就当用 Java 代码构 建 SQL 时避免了“丢失空间”的问题。

还有很多的注解项, 具体请文档查阅这里.

基于注解的事务管理

先简单的说一下什么是事务:

事务管理对于企业应用来说是至关重要的,即使出现异常情况,它也可以保证数据的一致性;

事务的定义:事务是指多个操作单元组成的合集,多个单元操作是整体不可分割的,要么都操作成功,要么都不成功.

其必须遵守4个原则:

  • 原子性:即事务是补课分隔的最小工作单元,事务内的操作要么全做,要么不做;
  • 一致性:在事务执行前数据库的数据处于正确的状态,而事务执行完成后数据库的数据还是应该处于正确的状态,即数据完整性约束没有被破坏;
  • 隔离性:并发事务执行间互不影响,在一个事务内部的操作对其他事务是不产生影响的,这需要事务隔离级别来指定隔离性;
  • 持久性:事务一旦执行成功,它对数据库的数据的改变必须是永久的,不会因任何故障导致数据不一致性;

事务的管理方式又分为两种:编程式事务管理 和 声明式事务管理;

编程式事务管理顾名思义就是在编写代码时直接手动操作事务,这样做就需要在业务逻辑代码中参杂事务管理的代码,显得很不友好;

而声明式的事务管理是基于注解/xml配置来做相关的事务声明,在spring中是建立在AOP之上的,其本质就是对方法代码进行拦截,然后在目标方法前加入一个事务,在执行完目标方法之后根据执行情况提交或者回滚事务,声明式事务的最大优点就是非侵入性的开发方式,使业务代码不受污染;

至于事务的 隔离级别 传播行为 超时 回滚规则等操作不在本文的探讨范围之内.

  1. 首先在springboot 启动类上添加注解:@EnableTransactionManagement 以开启事务注解.

    @SpringBootApplication
@MapperScan("com.zhangdi.mapper")//扫描mapper下所有的类
@EnableTransactionManagement
public class Application{

 public static void main(String[] args){
     SpringApplication.run(Application.class,args);
 }
}
  2. 在需要开启事务的 service类上添加@Transactional注解,添加在类上后,该类下的所有方法都开启事务,但是实际上我们只有 增/删/改 需要事务管理,所以还是建议在方法上添加事务注解.

UserMapperProvider:

public class UserMapperProvider{

    public String findByKeyWord(String keyWord){
        return new SQL(){
            {
                SELECT("*");
                FROM("user");
                WHERE("user_name like %" + keyWord + "%");
            }
        }.toString();

        //或是直接返回 sql String

        return "SELECT * FROM user WHERE user_name like %" + keyWord + "%";

        //又或者

        return new SQL().SELECT("*").FROM("user").WHERE("user_name like %" + keyWord + "%").toString();

    }

}

public String deletePersonSql() {

public String insertPersonSql() {
  String sql = new SQL()
    .INSERT_INTO("PERSON")
    .VALUES("ID, FIRST_NAME", "#{id}, #{firstName}")
    .VALUES("LAST_NAME", "#{lastName}")
    .toString();
  return sql;
}

public String selectPersonLike(final String id, final String firstName, final String lastName) {
  return new SQL() {{
    SELECT("P.ID, P.USERNAME, P.PASSWORD, P.FIRST_NAME, P.LAST_NAME");
    FROM("PERSON P");
    if (id != null) {
      WHERE("P.ID like #{id}");
    }
    if (firstName != null) {
      WHERE("P.FIRST_NAME like #{firstName}");
    }
    if (lastName != null) {
      WHERE("P.LAST_NAME like #{lastName}");
    }
    ORDER_BY("P.LAST_NAME");
  }}.toString();
}

  }}.toString();

public String updatePersonSql() {
  return new SQL() {{
    UPDATE("PERSON");
    SET("FIRST_NAME = #{firstName}");
    WHERE("ID = #{id}");
  }}.toString();
