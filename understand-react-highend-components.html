当我看到高阶组件(Higher-Order Components) 这个词时,这个牛逼闪闪的名词吧我镇住了,听了几位大师的讲解之后霍然开朗,这个牛逼闪闪的名词背后其实也是很简单的.

什么是 高阶组件

高阶组件就是一个函数,能够接受一个组件并返回一个新的组件;

在React中高阶组件是重用组件逻辑的一项高级技术.高阶组件并不是React API 的一部分,其源自 React 生态.

组件是将props转化成UI，然而高阶组件将一个组价转化成另外一个组件。

下面我们来拆解这个词, 高阶组件 可以理解为 高级的 组件.

先来说说什么是组件:React中定义,组件使你可以将UI划分为一个个的 独立的 可复用的 小部件,并可以对每个部件进行单独的设计.从定义上来说,组件就像是JavaScript的函数,组件可以接受任意输入(props),并返回React元素.

组件可以是 函数形式,也可以是类形式,但是我们又知道 JavaScript的 类(Class)可以看做是 一个语法糖,它的类写法只是让对象原型的写法更清晰,更像面向对象编程的语法而已,说到底 它还是Function;

以上,我们得出 组件 == 函数 故: 高阶组件 == 高阶函数;

接下来 看一个函数:

function sayHello(){
    var user = localStorage.getItem('user');
    console.log('hello' + user);
}
function sayBye(){
    var user = localStorage.getItem('user');
    console.log('Bye' + user);
}

上面两个函数 按照代码复用的原则可以这么改造:

function sayHello(user){
    console.log('hello' + user);
}
function sayBye(user){
    console.log('Bye' + user);
}

function higherFn(fn){
    var newFn = function(){
         var user = localStorage.getItem('user');
        fn(user);
    }
    return newFn;
}

从这个例子不难看出,我们做到了代码复用,符合高阶组件的定义接受一个函数,并返回一个新的函数,上面我们说了 组件也就是一个函数,那我们把上面的例子转化为 React 的组件 再来看看;

SayHello原始组件:

import React from 'react'

export default class SayHello extends React.Component{

    constructor(props){
        super(props);
        this.state = {
            user:''
        }
    }

    componentWillMount(){
        let user = localStorage.getItem('user');
        this.setState({
            user:user
        })
    }

    render(){
        return <div>hello {this.state.user}</div>
    }
}

SayBye原始组件:

import React from 'react'

export default class SayBye extends React.Component{

    constructor(props){
        super(props);
        this.state = {
            user:''
        }
    }

    componentWillMount(){
        let user = localStorage.getItem('user');
        this.setState({
            user:user
        })
    }

    render(){
        return <div>Bye! {this.state.user}</div>
    }
}

高阶组件:

import React from 'react'

export default higherComponent = (InnerComponent) => {
    class NewComponent extends React.Component{

        constructor(props){
            super(props);
            this.state = {
                user:''
            }
        }

        componentWillMount(){
            let user = localStorage.getItem('user');
            this.setState({
                user:user
            })
        }

        render(){
            return <InnerComponent usr={this.state.user}>
        }
    }
    return NewComponent
}

改造后的 SayHello,SayBye 组件:

import React from 'react'
import higherComponent from 'higherComponent'

class SayHello extends React.Component{
    render(){
        return <div>hello! {this.props.user}</div>
    }
}
//把组件传入 高阶组件 返回一个新的组件.
SayHello = higherComponent(SayHello)

export default SayHello

import React from 'react'
import higherComponent from 'higherComponent'

class SayBye extends React.Component{
    render(){
        return <div>Bye! {this.props.user}</div>
    }
}

//把组件传入 高阶组件 返回一个新的组件.
SayBye = higherComponent(SayBye)

export default SayBye

接着就引生出 react-redux的connect函数:

export default connect(mapStateToProps, mapDispatchToProps)(Component)

// connect是一个返回函数的函数（就是个高阶函数）
const enhance = connect(mapStateToProps, mapDispatchToProps);
// 返回的函数就是一个高阶组件，该高阶组件返回一个与Redux store
// 关联起来的新组件
const ConnectedComment = enhance(Component);

注意看mapStateToProps mapDispatchToProps,这两个名字已经明确了他们的含义:
把redux的state和action创建函数，通过props注入给了Component。

到这里，高阶组件就讲完了。再返回去理解下定义，是不是豁然开朗~
