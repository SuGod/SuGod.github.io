之前对webpack4.x做了一些学习。今天搭建一个相对完整的react的开发环境。废话不多说直接进入正题。

mkdir webpack4-react && cd webpack4-react && yarn init -Y
yarn add react react-dom
yarn add webpack webpack-cli webpack-dev-server -D

初始化并添加 react webpack 的相关依赖。原来 webpack 直接install webpack 就好，4.x 之后需要依赖webpack-cli。

配置方式

新建webpack.config.js这里要说下，webpack4.x版本引入了 mode 的概念，在运行 webpack 时需要指定使用 production 或 development 两个 mode 其中一个，这个功能也就是我们所需要的运行两套构建环境的能力。

webpack4.x官方文档上说， 除了导出单个配置对象，还有其他三种方式。

  1. 从 webpack 配置文件中导出一个函数。该函数在调用时，可传入两个参数：

    环境对象 env ，选项map 对象。

    env:

    map:这个对象描述了传递给 webpack 的选项，并且具有output-filename optimize-minimize等 key。

    即如下方式：

    module.exports = function(env, argv) {
    return {
        entry:'index.js'
        ...
    }
}

    ​

  2. 导出一个Promise对象

    该配置方式主要用于 异步加载配置所需的配置变量，webpack 将运行由配置文件导出的函数，并且等待 Promise 返回。

    module.exports = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve({
        entry: './app.js',
        /* ... */
      })
    }, 5000)
  })
}

    ​

  3. 导出多个配置对象

    作为导出一个配置对象/配置函数的替代，你可能需要导出多个配置对象（从 webpack 3.1.0 开始支持导出多个函数）。当运行 webpack 时，所有的配置对象都会构建。例如，导出多个配置对象，对于针对多个构建目标（例如 AMD 和 CommonJS）打包一个 library非常有用。

这里 后两种方式只做了解，使用第一种方式来进行配置，即导出函数的方式。

3.x版本时，消除 webpack.config.js 的环境差异，需要如下配置，加入NODE_ENV参数

{
  "scripts": {
    "build": "NODE_ENV=production webpack",
    "develop": "NODE_ENV=development webpack-dev-server"
  }
}

使用时通过process.env.NODE_ENV 来获取环境参数：

if (process.env.NODE_ENV === 'production') {
  // 生产环境需要做的事情..
}

4.x版本：

module.exports = function(env, argv) {
    return {
        mode: env.production ? 'production' : 'development',
        ...
    }
}

loader 及 plugin
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const ExtractTextPlugin = require('extract-text-webpack-plugin')
const autoprefixer = require('autoprefixer')
const webpack = require('webpack')
const mock = require('./mock/mock')

const paths = {
  appSrc: path.resolve(__dirname, 'src'),
  buildPath: path.resolve(__dirname, 'build'),
  appNodeModules: path.resolve(__dirname, 'node_modules')
}

module.exports = (env, argv) => ({
  entry: {
    main: path.resolve(__dirname, 'src/index.js')
  },
  output: {
    filename: '[name]-[hash].js',
    path: paths.buildPath,
    publicPath: '/'
  },
  module: {
    rules: [
      {
        test: /\.(js|jsx)$/,
        include: paths.appSrc,
        use: {
          loader: 'babel-loader',
          options: {
            compact: true,
          }
        }
      },
      {
        test: /\.(sass|scss)$/,
        exclude: paths.appNodeModules,
        use: [
          //4. style-loader 处理， 开启hmr
          { loader: 'style-loader', options: { hmr: true } },
          //3. css-loader 处理
          { loader: 'css-loader', options: { sourceMap: true } },
          //2. 使用 postcss 处理 css，根据浏览器环境的不同添加不同的前缀等处理。
          {
            loader: 'postcss-loader',
            options: {
              sourceMap: true,
              ident: 'postcss',
              plugins: () => [
                require('postcss-flexbugs-fixes'),
                autoprefixer({
                  browsers: [
                    '>1%',
                    'last 4 versions',
                    'Firefox ESR',
                    'not ie < 9',
                  ],
                  flexbox: 'no-2009',
                }),
              ],
            },
          },
          //1. 第一步 处理 sass
          { loader: 'sass-loader', options: { sourceMap: true } }
        ]
      },
      //处理图片
      {
        test: [/\.bmp$/, /\.gif$/, /\.jpe?g$/, /\.png$/, /\.svg$/],
        exclude: paths.appNodeModules,
        use: [
          {
            loader: 'url-loader',
            options: {
              limit: 1024    //小于1024kb的图片使用base64
            }
          }
        ]
      }
    ]
  },
  plugins: [
    //使用html-webpack-plugin生成html
    new HtmlWebpackPlugin({
      filename: 'index.html',
      template: path.resolve(__dirname, 'src/assets/template.html')
    }),
    //单独打包css,这个可以在 production 环境下配置。 
    //new ExtractTextPlugin('[name].css'),
      //dev-server 依赖插件
    new webpack.HotModuleReplacementPlugin()
  ],
  //配置 外部依赖 (runtime依赖) 参数:String/Array/Function/regex/object
  //使用时就可以 直接 import BMap from 'BMap'
  //jQuery 同理. 
  externals: {
    BMap: 'BMap'
  },
  //devserver 配置
  devServer: {
    publicPath: '/',
    contentBase: paths.buildPath,
    hot: true,
    hotOnly: true,
    inline: true,
    port: 3000,
    open: true,
    //使用代理，处理所有包含 /api 的请求 
    proxy: {
      '/api': {
        target: 'http://localhost:8001',
        secure: false
      }
    }
  }
})
模块热替换HMR(Hot Module Replacement)

这里要说明：仅仅在dev-server 配置中使用 hot:true 开启 HMR 是无效的。使用如下方式才能真正生效：

if (module.hot) {
  module.hot.accept('./library.js', function() {
    // 使用更新过的 library 模块执行某些操作...
  })
}

安装 react HMR 插件react-hot-loader：

yarn add react-hot-loader -D

使用：

import React from 'react'
import ReactDOM from 'react-dom'
import { AppContainer } from 'react-hot-loader'
import './assets/styles/global.scss'

import App from './App'


if (module.hot) {
  module.hot.accept(() => {
    render()
  })
}

(function render() {
  ReactDOM.render(<AppContainer><App /></AppContainer>, document.getElementById('root'))
})()

测试HMR正常，jsx 和 style/css/scss 都能触发 HMR。至于原理需要单独开一篇来讲。
