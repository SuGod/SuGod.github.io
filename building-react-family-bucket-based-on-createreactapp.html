什么是create-react-app?

create-react-app是Facebook官方推出的React应用开发脚手架工具,几乎可以零配置的构建webpack+react的开发环境,内置了热更新等一系列功能;

create-react-app 可以使用 yarn 替代 npm 来安装包.

常用命令:
npm start or yarn start          //启动
npm test or yarn test           //测试
npm run build or yarn build    //打包
npm run eject                 //弹出配置项

Yarn

  1. 安装yarn

windows 下载安装程序 Yarn 直接安装

Mac 使用brew安装

brew install yarn
  1. 基本命令
yarn add: 在你当前的包里添加一个包来使用。

yarn init: 初始化包的开发环境。

yarn install: 安装 package.json 文件里定义的所有依赖。

yarn publish: 发布一个包到包管理器。

yarn remove: 从你当前包里移除一个未使用的包。

运行 yarn 不带命令将运行 yarn install ，并把参数标志传递给它。

命令跟 npm 类似. 区别 add 时不需要 --save 默认保存到 package.json , 开发依赖 -dev

Quick Overview

//全局安装 create-react-app
npm install -g create-react-app

//新建一个 名为 my-app 的工程
create-react-app my-app

//构建完成之后 进入
cd my-app/

//启动
npm start

浏览器打开 http://localhost:3000/

生成的项目结构:

my-app
├── README.md
├── node_modules
├── package.json
├── .gitignore
├── public
│   └── favicon.ico
│   └── index.html
│   └── manifest.json
└── src
    └── App.css
    └── App.js
    └── App.test.js
    └── index.css
    └── index.js
    └── logo.svg
    └── registerServiceWorker.js

可以发现项目虽然是 基于 webpack 构建的 但是完全看不到熟悉的 webpack.config.js等配置文件,因为官方为了更优雅(?_?)的代码结构,就把配置项都通过 react-scripts 包配置好了.
在node_modules下找到 react-scripts可以看到 配置项都在这里.

打开 package.json 

{
  ....
  },
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test --env=jsdom",
    "eject": "react-scripts eject"
  }
}

实际开发中自定义配置在所难免,so 官方给了个 eject, 

npm run eject //弹出配置项

Note: this is a one-way operation. Once you eject, you can’t go back!
官方提示:这个操作是不可逆的,一旦操作就回不去了.

配置弹出来然后开始安装react全家桶

npm install --save react-router-dom         //4.*
npm install --save redux react-redux        //redux  react-redux
npm install --save redux-thunk              //异步中间件
npm install --save less-loader              //less-loader
npm install --save ....                     //各种包

在 config 目录下配置 webpack 如 less-loader url-loader 插件等.

配置ant-design

Ant Design 是一个致力于提升『用户』和『设计者』使用体验的中台设计语言。它模糊了产品经理、交互设计师、视觉设计师、前端工程师、开发工程师等角色边界，将进行 UE 设计和 UI 设计人员统称为『设计者』，利用统一的规范进行设计赋能，全面提高中台产品体验和研发效率。

安装antd

npm install --save antd   //或者 yarn add antd

使用: import antd 组件

在文件顶部引入 antd/dist/antd.css。

@import '~antd/dist/antd.css';

.App {
  text-align: center;
}

...

-------------------------------------------------------------

import React, { Component } from 'react';
import Button from 'antd/lib/button';
import './App.css';

class App extends Component {
  render() {
    return (
      <div className="App">
        <Button type="primary">Button</Button>
      </div>
    );
  }
}

export default App;

现在已经把组件成功运行起来了，但是在实际开发过程中还有很多问题，例如上面的例子实际上加载了全部的 antd 组件的样式（对前端性能是个隐患）。

使用 babel-plugin-import

//安装babel-plugin-import
npm install --save-dev babel-plugin-import //或者  yarn add babel-plugin-import --dev

打开 package.json

{
    ...
    "babel": {
    "presets": [
      "react-app"
    ],
    "plugins":[
      ["import", { "libraryName": "antd", "style": "css" }] //添加
    ]
    ...
  },
}

修改 App.js 再使用 Button组件就可以 import Button 就可以使用了.

  // scr/App.js
  import React, { Component } from 'react';
  import { Button } from 'antd';
  import './App.css';

  class App extends Component {
    render() {
      return (
        <div className="App">
          <Button type="primary">Button</Button>
        </div>
      );
    }
  }

  export default App;

其他 antd 配置可查看 antd文档
