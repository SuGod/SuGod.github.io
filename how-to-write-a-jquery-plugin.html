如何写一个jQuery插件

现在前端技术日新月异,Angular React Vue 三剑客是当前前端技术栈中最火热的框架.但是目前我们公司的技术栈还停留在 jQuery 打天下的段位,不过不得不说 jQuery 是一个非常非常非常非常非常(省略无数个)牛逼的 js库…

在一段时间的工作中发现,后端同学非常的依赖jQuery,但是在使用jQuery时会犯很多错误,如变量/function直接写在全局作用域下,为了所谓的代码复用,简单的抽出一些公共的部分写成一个全局的 function,做后每个人都写一套自己抽出的函数,造成页面引入大量名字不同但作用一样的所谓的工具类.然后就出现各种各样奇葩的 Bug..

为了解决这些问题,下面就来说说 如何写一个jQuery 插件吧;

首先要知道jQuery 最牛逼的地方就在于它的扩展性.扩展一个jQuery插件的方式有以下三种:

  1. 通过$.extend()来扩展jQuery
  2. 通过$.fn 向jQuery添加新的方法
  3. 通过$.widget()应用jQuery UI的部件工厂方式创建(能力有限我暂时还没使用过这种方式:(

第一种:$.extend()

$.extend({
    sayHello: function(name) {
        alert('hello' + name || 'world');
    }
})
$.sayHello(); //hello world!
$.sayHello('zhangsan'); //hello zhangsan

这种方式详单与直接在$上添加一个 sayHello的属性来扩展jQuery,相当于添加一个静态方法,这样做就完全不能使用jQuery强大的选择器..如果只是想添加静态方法这种方法是没有问题的.

第二种:通过$.fn 向jQuery添加新的方法

这种方法也是最常用的;

首先有人会问 $.fn 是什么?

使用过 jQuery的同学都应该知道 $ === jQuery,$([selector]) 和 jQuery([selector])完全一样.

阅读jQuery源码会发现. 

jQuery.fn = jQuery.prototype = {....}
//其中 jQuery == $ so:
jQuery.fn == $.fn == $.prototype

然后我们来看一下它的基本格式:

;(function($,window,document,undefined){

    $.fn.pluginName = function(options){
        var defaulats = {};
        var opts = $.extend({},defaults,options);
        //fn..
    }

})(jQuery,window,document)

//调用插件
$(selector).pluginName(options);

下面我们来解释为什么这么写.

  1. ; 可以避免在执行我们写的代码前引用了他人写的代码最后没写;造成错误.
  2. 一个自调用的function,接收4个参数 $,window,document,undefined 底部传入 3个参数:jQuery,window,document
    1. 首先解释为什么传入3个参数却要接受4个参数,多了个 undefined,它是js预定义的一个全局变量，他的值就是’undefined’。在ES3中，有一个bug,undefined 是一个可读可写的变量,可以给他赋任意值,但到了ES5就被修复了,他是一个只读的值,所以虽然没有报错,但是赋值语句是没有效果的.这么做的目的就是为了避免undefined不是我们想要的undefined;
    2. 同样 传入 jQuery,window,document 也是防止 这些变量被改写或被污染;
    3. 自调用函数,是为了避免污染全局作用域;这样我们不管有多少个插件 他们内部的 变量都是在内部使用 不会影响其他插件的使用;
  3. $.fn.pluginName = function(){} 中 pluginName就是我们的插件的名称;就是把我们的插件挂载在jquery的原型链上,这样我们就可以使用jQuery的一切方法.
  4. options参数 和 defauls 是为了方便的自定义插件的参数,通过 使用 $.extend()方法来合并 我们调用插件时传入的参数和插件的默认参数,同时返回一个新的opts 这样做不会覆盖掉传入的 参数对象 和 默认参数对象.

然后我们来写一个简单的插件:

;(function($,window,document,undefined){

    $.fn.myPlugin = function(options){
        var defaults = {
            color:'red',
            fontSize:'20px'
        };
        var opts = $.extend({},defaults,options || {});
        //这里的 this 就是jQuery选择器选中的元素集合.

        return this.each(function(){
            //这里的this 是遍历后 当前元素 this
            $(this).css(opts);
            //或
            /*$(this).css({
                color:opts.color,
                fontSize:opts.fontSize
            });*/
        });

    }

})(jQuery,window,document)

//调用插件
$('p').myPlugin({
    color:'blue'
});

这样我们就把页面上所有的 p 元素 颜色设置为蓝色,字号设置为20px;如果不传入任何参数就把所有p元素颜色设置为红色字号20px;

代码中 return 返回当前选择器的 this 对象, 是为了能够链式调用 .

$('p').myPlugin().css('border','1px solid #ccc');

或者使用构造函数的写法:

;(function($,window,document,undefined){

    //定义构造函数.
    var MyPlugin = function(ele, opts) {
        this.$element = ele,
        this.defaults = {
            color: 'red',
            fontSize: '20px',
            textDecoration: 'none'
        },
        this.options = $.extend({}, this.defaults, opts)
    }
    //定义方法
    MyPlugin.prototype = {
        beautify: function() {
            return this.$element.css({
                color: this.options.color,
                fontSize: this.options.fontSize,
                textDecoration: this.options.textDecoration
            });
        }
    }

    $.fn.myPlugin = function(options) {
        //创建插件实例
        var myPlugin = new MyPlugin(this, options);
        //调用其方法
        return myPlugin.beautify();
    }

})(jQuery,window,document);

//使用方法和之前的一样.
