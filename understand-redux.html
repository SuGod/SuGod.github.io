理解Redux

React 是 DOM 层的一个抽象层,并不是 Web 应用的完整解决方案.
有两个方面没有涉及:

  • 代码结构
  • 组件之间通信

当我还没搞明白redux 到底是个啥东西的时候,大师说了这句:

“如果你不知道是否需要 Redux ,那就是不需要它”.

WTF?
当我第一次尝试使用react做项目时,翻了一天文档,写了几个demo后 觉得 react so easy…我也真的不知道我是否需要 Redux.听大师的就是不需要..
结果当项目复杂度提升后,因为组件间通信的问题,搞的焦头烂额!甚至一度想过放弃react..这特么什么玩意? 单向数据流? 怎么通信? 然后开始把注意力转向 Angular..因为双向数据绑定.Form 等一切都是那么美好.
然而大师又说:

“只有遇到 React 实在解决不了的问题,你才需要 Redux “

我仿佛懂了,我需要Redux..!

接着看到阮一峰大神的博客,他列举了几种场景:
不需要:

  • 用户的使用方式非常简单
  • 用户之间没有协作
  • 不需要与服务器大量交互,也没有使用 WebSocket
  • 视图层（View）只从单一来源获取数据

需要:

  • 用户的使用方式复杂
  • 不同身份的用户有不同的使用方式（比如普通用户和管理员）
  • 多个用户之间可以协作
  • 与服务器大量交互,或者使用了WebSocket
  • View要从多个来源获取数据

考虑使用:

  • 某个组件的状态,需要共享
  • 某个状态需要在任何地方都可以拿到
  • 一个组件需要改变全局状态
  • 一个组件需要改变另一个组件的状态

阮一峰说 Redux 的核心思想:

  • web应用是个状态机,视图与状态是一一对应的;
  • 所有的状态,保存在一个对象里;

然后我说说我的理解吧.

先说一下组件通信:比如组件A 需要根据 组件B 的一些操作来改变自身,或者根据组件A的状态来改变自己的状态.这时就需要组件A改变时通知组件B.甚至还有组件C会依赖组件B…..

react的优势同时也是痛点 单向数据流 数据从父组件一级一级的往下传递,组件嵌套多层后,数据的传递就非常复杂,组件间的通信也需要通过一个桥梁,公共父组件来传递消息.如果两个组件是兄弟组件还好说,如果不是,可以想想这么做有多痛苦.

Redux 就是来解决这些问题的,解决的核心思想就是所有状态保存在一个对象里.,

它带来了几个概念:

store action reducer

1. Store

所有状态保存在一个对象里.这个对象就是 store

Store对象包含所有数据.如果想得到某个时点的数据,就要对 Store生成快照.这种时点的数据集合,就叫做 State.

Redux 规定, 一个 State 对应一个 View.只要 State 相同,View 就相同.你知道 State,就知道 View 是什么样,反之亦然.

2. Action

State 的变化,会导致 View 的变化.但是,用户接触不到 State,只能接触到 View.所以,State 的变化必须是 View 导致的.Action 就是 View 发出的通知,表示 State 应该要发生变化了.
可以这样理解,Action 描述当前发生的事情.改变 State 的唯一办法,就是使用 Action.它会运送数据到 Store.

3. Action Creator

定义一个函数来生成 Action ,这个函数就叫 Action Creator,为什么需要这个? 看下面.

4. store.dispatch()

store.dispatch()是 View 发出 Action 的唯一方法.

//默认写法
store.dispatch({type:'ADD',payload:'需要携带的数据..'});
//type:'ADD',payload:'需要携带的数据..'} 就是action 每次都这么写会很麻烦,so 定义function来生成action
const ADD = 'ADD';
const addTodo = (text) => {type: ADD,text};

store.dispatch(addTodo('需要携带的数据..'));

6. Reducer

Store 收到 Action 以后,必须给出一个新的 State,这样 View 才会发生变化.这种 State 的计算过程就叫做 Reducer.

//reducer 是一个函数,它接受 Action 和当前 State 作为参数,返回一个新的 State
const reducer = (state = 0, action) => {
    //store 收到action 后 根据 action.type 来判断如何计算state
  switch (action.type) {
    case 'ADD':
    //必须返回一个新的state 不能直接操作state
      return state + action.payload;
    default: 
      return state;
  }
};

const ADD = 'ADD';
//action creator
const addTodo = (text) => {type: ADD,payload:text};

总结起来说就是: 要想改变store 只有发送 action , 要想发出action 只有通过store.dispatch(), 改变的具体实现就是 reducer 生成action可以通过action creator.

end.
